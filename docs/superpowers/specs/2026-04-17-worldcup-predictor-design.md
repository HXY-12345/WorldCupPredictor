# 世界杯比分预测智能体 - 设计文档

**日期**: 2026-04-17
**状态**: 待实施

---

## 1. 项目概述

### 1.1 目标
构建一个世界杯比分预测智能体，为广大球迷提供实时赛程、比分展示和 AI 智能预测功能。

### 1.2 核心功能
- 实时展示世界杯赛程和比分
- AI 预测未来比赛结果
- 展示比赛详细数据（历史交锋、球队排名、球员状态）
- 自动数据刷新（30秒轮询）

---

## 2. 设计决策记录

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 目标用户 | 广大球迷 | 需要友好的界面和实时信息 |
| 数据来源 | 爬虫抓取 | 灵活获取多源数据 |
| 前端技术 | HTML+CSS+JS | 轻量、简单、无需构建 |
| 后端技术 | Python FastAPI | 快速开发、类型安全、异步支持 |
| AI 预测 | 直接调用大模型 API | 简单可靠、无需训练 |
| 页面布局 | 分组日期布局 | 适合多天赛程浏览 |
| 卡片详情 | 丰富模式 | 提供充分上下文信息 |
| 视觉风格 | 经典体育风格 | 蓝绿色调，专业可信 |
| 预测展示 | 比分卡（记分牌） | 视觉冲击力强 |
| 更新策略 | 定时轮询 30秒 | 简单可靠 |

## 5. API 设计

### 5.1 接口列表

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/matches` | 获取所有比赛 |
| GET | `/api/matches/{match_id}` | 获取单场比赛详情 |
| POST | `/api/predict/{match_id}` | 触发 AI 预测 |
| POST | `/api/refresh` | 手动触发爬虫更新 |
| GET | `/api/health` | 健康检查 |

### 5.2 接口详情

#### GET /api/matches

**响应示例:**
```json
{

  "total": 48
```

## 8. 爬虫设计

### 8.1 数据源优先级

1. **FIFA 官网** - 主数据源
2. **ESPN / Flashscore** - 备用数据源
3. **Transfermarkt** - 球队/球员信息补充

### 8.2 爬虫策略

- 使用 `httpx` 异步请求
- 请求头伪装（User-Agent）
- 限流控制（避免被封）
- 多源备份（主源失效时切换）
- 本地缓存（减少请求）

### 8.3 更新频率

| 数据类型 | 更新频率 |
|----------|----------|
| 赛程数据 | 每日 1 次 |
| 比分数据 | 进行中比赛每 5 分钟 |
| 球队信息 | 每日 1 次 |

## 12. 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 爬虫被封 | 多数据源备份 + 请求限流 |
| API 调用成本 | 预测结果缓存 24 小时 |
| 实时性不足 | 赛中增加轮询频率 |
| 数据准确性 | 交叉验证多源数据 |
