目录结构

worldcup/
├── frontend/              # 前端 (纯 HTML/CSS/JS)
│   ├── index.html         # 主页
│   ├── css/
│   │   ├── reset.css      # 重置样式
│   │   ├── variables.css  # 颜色变量 (蓝绿体育风)
│   │   └── style.css      # 主样式
│   └── js/
│       ├── api.js         # API 调用
│       ├── ui.js          # UI 交互
│       └── app.js         # 主入口
│
├── backend/               # 后端 (FastAPI + Python)
│   ├── main.py            # FastAPI 应用入口
│   ├── api/               # API 路由
│   │   ├── matches.py     # 赛程接口
│   │   └── predict.py     # 预测接口
│   ├── crawler/           # 爬虫模块
│   │   ├── base.py        # 基础爬虫类
│   │   └── sources/       # 各数据源爬虫
│   ├── ai/                # AI 模块
│   │   ├── client.py      # 大模型 API 客户端
│   │   └── prompts.py     # 预测提示词模板
│   └── data/              # 数据存储
│       ├── matches.db     # SQLite 数据库
│       └── cache/         # 预测结果缓存
│
└── requirements.txt       # Python 依赖

核心功能模块

前端功能

  • 分组日期赛程展示（可展开/折叠）
  • 丰富模式比赛卡片
  • 记分牌式 AI 预测展示
  • 30 秒自动轮询刷新
  • 手动刷新按钮
  • 加载状态与错误提示

后端功能

  • RESTful API 接口
  • 定时爬取赛程比分数据
  • 调用大模型生成预测
  • 预测结果缓存（避免重复调用）
  • CORS 跨域支持
  • 数据验证与错误处理

API 设计（草案）

GET /api/matches
→ 返回所有比赛数据（赛程+比分+预测）
POST /api/predict/:match_id
→ 触发 AI 预测指定比赛
POST /api/refresh
→ 手动触发爬虫更新数据
GET /api/health
→ 健康检查
