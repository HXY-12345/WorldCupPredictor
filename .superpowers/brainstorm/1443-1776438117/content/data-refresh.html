数据更新策略

选择获取最新赛程和比分的方式

A

定时轮询

前端每隔一段时间自动请求后端获取最新数据

✓ 优点

  • 实现简单
  • 兼容性好
  • 无状态，易扩展

✗ 缺点

  • 有延迟（轮询间隔）
  • 可能产生无效请求
示例 每 30 秒请求一次后端接口
B

SSE 推送

后端有新数据时主动推送给前端

✓ 优点

  • 实时性强
  • 节省带宽
  • 自动重连

✗ 缺点

  • 需要保持连接
  • 实现稍复杂
示例 EventSource 连接，有比分变化立即推送
C

手动刷新

用户主动点击刷新按钮获取最新数据

✓ 优点

  • 最简单
  • 无服务器压力
  • 用户可控

✗ 缺点

  • 不是真正的实时
  • 需要用户操作
示例 点击 "刷新" 按钮更新数据
